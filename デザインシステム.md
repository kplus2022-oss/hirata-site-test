# デザインシステム

対象ファイル: `index.html`

このドキュメントは、現在の `index.html` に実装されている世界観とUIルールを固定し、今後の修正・追加でも同じデザインスタイルを維持するための運用基準です。

## 1. デザインコンセプト

- 方向性: 静けさ、余白、思想性、上質さ、和の気配を持つエディトリアルデザイン
- 印象: 豪華ではなく静謐。派手ではなく深い。情報サイトではなく「思想と体験を読む場」
- キーワード:
  - 静かな高級感
  - 日本的な余白感
  - 詩的なタイポグラフィ
  - 低彩度の写真表現
  - ガラスのような半透明レイヤー
  - 緩やかな動き

このサイトは、一般的なLPやSaaSサイトのような「強いCTA中心のUI」に寄せないこと。

## 2. ブランドトーン

- 声の質感: 落ち着いている、誠実、内省的、断定しすぎない
- 情報の見せ方: 急がせない、詰め込まない、読ませる
- 視線誘導: 強い色で押すのではなく、余白・文字組み・写真の重なりで導く

## 3. カラーパレット

`index.html` のCSS変数を基準とする。

### 基本色

- 背景色: `#F7F6F2`
- ガラス背景: `rgba(255, 255, 255, 0.85)`
- 本文色: `#2A2825`
- 補助文字色: `#7A7875`
- 境界線: `#E2E0DB`

### 運用ルール

- ベース背景は暖かみのある生成り系を維持する
- 真っ白 `#FFFFFF` を大面積の地色にしない
- 真っ黒 `#000000` を本文の基本色にしない
- アクセントカラーを新規導入しない
- 色の主役は「背景の温度差」と「写真の空気感」で作る

## 4. タイポグラフィ

### 使用フォント

- 日本語本文・見出し: `Shippori Mincho`
- 英字ディスプレイ・装飾見出し: `Cormorant Garamond`

### 文字組みの原則

- 見出しはやや大きく、余白で見せる
- 本文は読みやすさ優先で、詰めすぎない
- 英字ラベルは大文字・広めの字間を使う
- 日本語本文は行間をしっかり確保する

### 推奨ルール

- 英字のセクションラベル: `tracking-[0.28em]` 前後
- セクション見出し: 和文主体、必要に応じて英字を添える
- 本文: 長文では `line-height` を大きめに保つ
- 数字・記号を多用して情報的に見せすぎない

## 5. レイアウトシステム

### 全体構成

このサイトの基本構成は以下を維持する。

1. 固定ヘッダー
2. ヒーロー
3. メッセージ導入
4. ナビゲーションカード群
5. 詳細セクション群
6. コンタクト
7. フッター

### コンテナ幅

- 主コンテンツ幅は広すぎず、読みやすさを優先する
- セクションごとに「余白で見せる」設計を維持する
- 画面端いっぱいに文字を置かない

### 余白ルール

- セクション間は大きめに取る
- 同一セクション内は「見出し」「本文」「画像」「補足情報」の間隔を整理する
- モバイルでも余白を削りすぎない

## 6. ビジュアルルール

### 写真

- 低彩度、自然光、空気感のある写真を使う
- 過度に広告的な写真は避ける
- 医療・教育・思想・暮らしに接続する静かな写真を優先する
- 写真の役割は説明ではなく情緒補強

### 画像の見せ方

- 角丸は使いすぎない
- 枠線や余白で静かに整える
- 画像の上にガラスパネルやテキストを重ねる場合は、読みやすさを優先する
- 彩度・コントラストは強くしすぎない

### 動画

- ヒーロー動画は雰囲気の核として使う
- 無音、ループ、自然な動きが前提
- 動きが強すぎる素材は避ける
- 上に重ねる文字が読みづらい場合は、薄いグラデーションやぼかしで補正する

## 7. モーションルール

### 基本姿勢

- 動きは演出ではなく呼吸として扱う
- 速くしない
- 数を増やしすぎない
- UIアニメーションより、スクロール連動の奥行き表現を重視する

### 現在の実装方針

- `reveal` によるフェード＋ぼかし解除
- ヒーローのゆるやかなズーム
- パララックス背景
- スクロールインジケーターの線アニメーション
- タイムライン進行表現

### 維持ルール

- 新しいアニメーションを足す場合も、0.4秒〜1.6秒程度の落ち着いたレンジに収める
- バウンド、過剰なスケール、派手な回転は使わない
- hover は軽く、品位を崩さない程度に留める

## 8. コンポーネント規則

### 8-1. ヘッダー

- 透明感のある固定ヘッダーを基本とする
- 背景は完全ベタ塗りにしない
- ロゴ、主要ナビ、予約/連絡導線は整然と配置する
- PCでは余白を広めに、モバイルでは情報密度を抑えて整理する

### 8-2. ヒーロー

- サイトの思想を最も強く伝える場所
- 背景は写真または動画を全面で使う
- テキストは中央寄せまたは静かな重ね構成を維持する
- 主見出しは説明文ではなく、余韻のある短い言葉を優先する
- サブテキストは行間を取り、読ませる設計にする

#### ヒーローでやってはいけないこと

- 派手なグラデーション背景に差し替える
- CTAボタンを主役にする
- 文字量を増やしすぎる
- SaaS風の左テキスト右画像構成に変える

### 8-3. セクション導入

- 英字ラベル + 和文タイトルの組み合わせを基本とする
- 英字は装飾ではなく、視線の整流として機能させる
- 見出し下には必ず十分な余白を置く

### 8-4. ナビゲーションカード

- 各テーマへの入口として機能させる
- 画像、英字コード、和文タイトルの秩序を保つ
- クリック誘導を強く見せすぎない
- カードは均一に並べるが、余白で呼吸を作る

### 8-5. ガラスパネル

- 半透明の白背景 + ぼかし + 細い境界線を基本とする
- 強い影を使わない
- 情報を浮かせるための補助UIとして使う
- 多用しない

### 8-6. ボタン

- 枠線主体、または静かな塗りを基本とする
- 色で主張しない
- 角丸は控えめ
- ホバー時の変化も小さく抑える

### 8-7. フッター

- 情報整理の場として設計する
- 装飾を増やしすぎない
- 本文より少し軽い扱いで、サイト全体の静けさを保つ

## 9. セクション追加時のルール

新しいセクションを追加する場合は、以下を満たすこと。

1. 英字ラベル + 和文見出しの構成に合わせる
2. 背景色は既存トーンから外さない
3. 文字サイズを既存見出し体系から逸脱させない
4. 写真を使う場合は低彩度・自然光・静けさを優先する
5. CTA中心ではなく、読み物中心の構成にする
6. セクション上下の余白を既存のリズムに合わせる
7. 1セクション内に要素を詰め込みすぎない

## 10. 修正時の禁止事項

以下の変更は、現行デザインシステムを壊しやすいため原則禁止とする。

- 青・赤・緑などの強いアクセント色の追加
- ゴシック体主体への変更
- 角丸カードや派手なシャドウの多用
- SaaSテンプレート風のヒーローへの変更
- 情報量を一気に増やす改修
- ボタンやバナーを目立たせすぎる改修
- アニメーションを増やしすぎる改修
- 余白を削って密度を上げる改修

## 11. 実装トークンの基準

今後CSSを整理する場合も、以下のトークンは維持すること。

```css
:root {
  --bg-color: #F7F6F2;
  --bg-white: rgba(255, 255, 255, 0.85);
  --text-main: #2A2825;
  --text-light: #7A7875;
  --border-color: #E2E0DB;
}
```

フォントも以下を基準とする。

```css
font-family: 'Shippori Mincho', serif;
font-family: 'Cormorant Garamond', serif;
```

## 12. 今後の運用方法

### 修正前

- まずこの `デザインシステム.md` を確認する
- 追加したい内容が「情報追加」なのか「世界観変更」なのかを分けて判断する
- 世界観変更でない限り、配色・書体・余白ルールは固定する

### 修正時

- 既存セクションの構成ルールを流用する
- 新規UIは作りすぎず、既存パターンの再利用を優先する
- 迷った場合は「目立たせる」ではなく「静かに整える」を選ぶ

### 修正後

以下を必ず確認する。

1. 文字組みが既存と揃っているか
2. 余白が詰まりすぎていないか
3. 写真のトーンが浮いていないか
4. CTAや装飾が強くなりすぎていないか
5. モバイルで静けさが保たれているか

## 13. 判断基準

デザイン判断に迷ったときは、以下の優先順位で決める。

1. 読みやすいか
2. 静かな高級感を保てているか
3. 既存セクションと同じ空気感か
4. 情報を足しすぎていないか
5. 動きが過剰になっていないか

## 14. 一言で定義するこのサイトのデザイン

「思想を読むための、静かで上質な和モダン・エディトリアルサイト」
