# AGENTS.md

このフォルダ内のWeb制作物を修正・追加するときは、必ず [デザインシステム.md](./デザインシステム.md) を先に確認し、そのルールを優先すること。

## 最優先ルール

- 配色は `index.html` の既存トークンを維持する
- 書体は `Shippori Mincho` と `Cormorant Garamond` を維持する
- サイトの方向性は「静かで上質な和モダン・エディトリアル」を崩さない
- 一般的なSaaSテンプレートや広告LPのUIに寄せない
- 強いアクセントカラーを追加しない
- 情報を詰め込みすぎない
- CTAを主役にしない
- 余白を削って密度を上げない

## 修正前に必ず確認すること

1. 今回の変更は「情報追加」か「世界観変更」かを切り分ける
2. 世界観変更でない限り、色・書体・余白・写真トーンは固定する
3. 新規セクションを作る場合は、既存セクションの構成を流用する
4. 新しいUIパターンを増やす前に、既存パターンの再利用を優先する

## 実装ルール

- 既存のCSS変数を優先して使う
- 色や余白の値を直接ハードコードする前に、既存トークン化できないか確認する
- 画像は低彩度・自然光・静けさのあるものを優先する
- アニメーションは遅めで控えめにする
- hoverや演出は目立たせるためではなく、空気感を整えるために使う

## 追加時の禁止事項

- 青・赤・緑などの強い新規アクセント色
- ゴシック主体のUIへの変更
- 派手なシャドウや丸すぎるカード
- 左テキスト右画像のテンプレLPヒーロー
- ボタンだらけの情報設計
- 動きすぎるパララックスや過剰なアニメーション

## 完了前チェック

- 文字組みは既存と揃っているか
- 余白の呼吸感は保てているか
- 写真や動画のトーンは浮いていないか
- モバイルで空気感が壊れていないか
- 今回の変更で必要なら `デザインシステム.md` も更新したか
